*Предыдущее задание*

****Организовать взаимодействие микросервисов посредством REST****

**Цель: использовать REST для взаимодействия микросервисов**

Результат: приложение с использованием RestTemplate в клиентской части

Проверка работы:
1. Запустить приложение RestServerApplication - это сервер, к которому будет обращаться клиент по REST
2. Запустить приложение RestClientApplication - это клиент, который обращается к серверу по REST. 
3. В консоли клиента будут отображены результаты запросов. Ошибок быть не должно. Выполняются запросы добавления, чтения, 
записи и удаления объектов. Также проверяется использование кеширования объектов.

*Текущее задание*

****Обернуть приложение в docker-контейнер.****

**Цель**

Деплоить приложение в современном DevOps-стеке.

Проверка работы:
1. Осуществить сборку jar-файлов приложений rest-client-0.0.1-SNAPSHOT.jar и rest-server-0.0.1-SNAPSHOT.jar командой mvnw.cmd package
2. Перейти в каталог docker
3. Скопировать файлы из target в каталог docker командой copy.cmd
4. Запустить приложения RestServerApplication и RestClientApplication в контейнерах Docker командой docker compose up -d
5. В браузере проверить работу приложений http://localhost:8080/client/test - ответ должен содержать OK
6. Убедиться в отсутствии ошибок и наличия информационных сообщений в журналах docker desktop View Details приложения rest-client
